# Business Rules — Unit 1: Backend

## 1. 인증/인가 규칙

### BR-AUTH-01: 관리자 로그인 시도 제한
- 연속 3회 실패 시 5분 잠금
- 잠금 해제 후 실패 횟수 리셋
- 성공 로그인 시 실패 횟수 즉시 리셋

### BR-AUTH-02: JWT 토큰 정책
- 관리자 토큰 유효기간: 16시간
- 테이블 토큰 유효기간: 16시간
- Payload: user_type(admin/table), id, store_id, role(admin만), exp

### BR-AUTH-03: 역할 기반 접근 제어
| 기능 | 매니저 | 직원 | 테이블 |
|------|--------|------|--------|
| 주문 모니터링 | ✅ | ✅ | ❌ |
| 주문 상태 변경 | ✅ | ✅ | ❌ |
| 주문 삭제 | ✅ | ❌ | ❌ |
| 메뉴 관리 (CRUD) | ✅ | ❌ | ❌ |
| 테이블 관리 | ✅ | ❌ | ❌ |
| 이용 완료 처리 | ✅ | ❌ | ❌ |
| 과거 내역 조회 | ✅ | ❌ | ❌ |
| 메뉴 조회 | ✅ | ✅ | ✅ |
| 주문 생성 | ❌ | ❌ | ✅ |
| 주문 내역 조회 (본인 세션) | ❌ | ❌ | ✅ |

### BR-AUTH-04: 비밀번호 저장
- bcrypt 해싱 (cost factor: 12)
- 평문 비밀번호 저장 금지

## 2. 주문 규칙

### BR-ORD-01: 주문 상태 전이
- 허용: pending ↔ preparing, pending → completed, preparing → completed
- 금지: completed → 다른 상태 (되돌리기 불가)

### BR-ORD-02: 주문 생성 조건
- 최소 1개 이상의 메뉴 항목 필수
- 모든 메뉴가 동일 매장 소속이어야 함
- 모든 메뉴가 is_available=TRUE여야 함
- 수량은 1 이상의 정수

### BR-ORD-03: 주문 금액 계산
- OrderItem.subtotal = quantity × unit_price (주문 시점 가격 스냅샷)
- Order.total_amount = SUM(OrderItem.subtotal)
- 메뉴 가격 변경이 기존 주문에 영향 없음 (스냅샷)

### BR-ORD-04: 주문 삭제 (Soft Delete)
- is_deleted=TRUE 설정 (데이터 보존)
- 삭제된 주문은 목록 조회에서 제외 (is_deleted=FALSE 필터)
- 삭제 시 세션 총액에서 해당 금액 차감
- 매니저 권한만 삭제 가능

### BR-ORD-05: 주문 번호
- 형식: YYYYMMDD-XXXX (매장별 당일 순번)
- 매장별 독립 채번
- 날짜 변경 시 순번 리셋

## 3. 테이블 세션 규칙

### BR-SES-01: 세션 시작
- 첫 주문 생성 시 자동 시작 (활성 세션 없을 때)
- 테이블당 동시 활성 세션 최대 1개

### BR-SES-02: 세션 종료 (이용 완료)
- 관리자(매니저)만 이용 완료 처리 가능
- 이용 완료 시:
  1. 해당 세션의 비삭제 주문을 OrderHistory에 복사
  2. 원본 Order 데이터 유지 (삭제하지 않음)
  3. TableSession.is_active = FALSE
  4. Table.current_session_id = NULL
- 활성 세션 없는 테이블은 이용 완료 불가 (400 에러)

### BR-SES-03: 세션 총액
- 주문 생성 시 총액 증가
- 주문 삭제(soft) 시 총액 감소
- 이용 완료 시 총액은 세션에 기록된 채로 보존

### BR-SES-04: 고객 주문 조회 범위
- 현재 활성 세션의 주문만 표시
- is_deleted=FALSE인 주문만 표시
- 이전 세션 주문은 고객에게 표시하지 않음

## 4. 메뉴 관리 규칙

### BR-MENU-01: 메뉴 필수 필드
- 메뉴명: 필수, 1~100자
- 가격: 필수, 0 이상 정수 (무료 가능, 상한 없음)
- 카테고리: 필수, 동일 매장 소속 카테고리
- 설명: 선택, 최대 1000자
- 이미지 URL: 선택, 최대 500자

### BR-MENU-02: 메뉴 삭제
- 메뉴 삭제 시 기존 주문의 OrderItem에는 영향 없음 (스냅샷 보존)
- 삭제된 메뉴로 신규 주문 불가

### BR-MENU-03: 노출 순서
- sort_order 값으로 정렬 (오름차순)
- 동일 sort_order 시 생성 시각 순

### BR-MENU-04: 카테고리 삭제
- 해당 카테고리에 메뉴가 있으면 삭제 불가 (400 에러)
- 메뉴를 먼저 다른 카테고리로 이동하거나 삭제 후 카테고리 삭제

## 5. 데이터 검증 규칙

### BR-VAL-01: 공통 검증
- 문자열 필드: 앞뒤 공백 제거 (trim)
- ID 필드: 양의 정수
- 날짜 필드: ISO 8601 형식

### BR-VAL-02: 가격 검증
- 정수 타입 (원 단위)
- 최소: 0 (무료 가능)
- 최대: 제한 없음

### BR-VAL-03: 수량 검증
- 정수 타입
- 최소: 1
- 최대: 99 (합리적 상한)
