# 테이블오더 서비스 요구사항

## 의도 분석 (Intent Analysis)

- **사용자 요청**: 테이블오더 서비스 신규 구축
- **요청 유형**: New Project (Greenfield)
- **범위 추정**: System-wide — 고객용 UI, 관리자 UI, API 서버, 데이터베이스 전체 구축
- **복잡도 추정**: Complex — 실시간 통신(SSE), 세션 관리, 인증, 다중 역할 등 포함

## 2. 기능 요구사항

### 2.1 고객용 기능 (Customer)

#### FR-C01: 테이블 태블릿 자동 로그인 및 세션 관리
- 관리자가 1회 초기 설정 수행 (매장 식별자, 테이블 번호, 테이블 비밀번호)
- 로그인 정보 로컬 저장 후 자동 로그인
- 접근 방식: QR 코드 스캔 + 태블릿 고정 설치 모두 지원

#### FR-C02: 메뉴 조회 및 탐색
- 메뉴 화면이 기본 화면으로 항상 표시
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세 정보: 메뉴명, 가격, 설명, 이미지
- 카테고리 간 빠른 이동
- 카드 형태 레이아웃, 터치 친화적 버튼 (최소 44x44px)

#### FR-C03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절
- 총 금액 실시간 계산
- 장바구니 비우기
- 클라이언트 측 로컬 저장 (새로고침 시 유지)
- 서버 전송은 주문 확정 시에만

#### FR-C04: 주문 생성
- 주문 내역 최종 확인 후 주문 확정
- 주문 성공 시: 주문 번호 표시 → 장바구니 비우기 → 5초 후 메뉴 화면 자동 리다이렉트
- 주문 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장 ID, 테이블 ID, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID

#### FR-C05: 주문 내역 조회
- 현재 테이블 세션 주문만 표시 (이전 세션 제외)
- 주문 시간 순 정렬
- 주문별 상세: 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)
- 매장 이용 완료 처리된 주문 제외

### 2.2 관리자용 기능 (Admin)

#### FR-A01: 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호 입력
- 16시간 세션 유지 (JWT 토큰)
- 브라우저 새로고침 시 세션 유지
- 16시간 후 자동 로그아웃
- 비밀번호 bcrypt 해싱
- 로그인 시도 제한
- 다수 관리자 계정 + 역할 구분 (매니저/직원)

#### FR-A02: 실시간 주문 모니터링
- SSE 기반 실시간 주문 목록 업데이트
- 그리드/대시보드 레이아웃: 테이블별 카드 형태
- 각 테이블 카드: 총 주문액, 최신 주문 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경 (대기중/준비중/완료)
- 신규 주문 시각적 강조 (색상/애니메이션)
- 2초 이내 주문 표시
- 테이블별 필터링

#### FR-A03: 테이블 관리
- **초기 설정**: 테이블 번호/비밀번호 설정, 16시간 세션 생성
- **주문 삭제**: 확인 팝업 → 즉시 삭제 → 총 주문액 재계산
- **테이블 세션 처리**: 이용 완료 시 주문 내역을 과거 이력으로 이동, 현재 주문/총액 리셋
- **과거 주문 내역 조회**: 테이블별 과거 주문 목록 (시간 역순), 날짜 필터링

#### FR-A04: 메뉴 관리
- 메뉴 CRUD (조회/등록/수정/삭제)
- 메뉴 정보: 메뉴명, 가격, 설명, 카테고리, 이미지 URL
- 메뉴 노출 순서 조정
- 필수 필드 검증, 가격 범위 검증
- MVP에서는 이미지 URL만 지원 (향후 업로드 기능 추가 예정)

## 3. 비기능 요구사항

### NFR-01: 성능
- SSE 기반 실시간 주문 알림: 2초 이내 표시
- 소규모 환경 최적화: 단일 매장, 테이블 20개 이하

### NFR-02: 보안
- JWT 토큰 기반 인증 (관리자 16시간, 테이블 16시간)
- 비밀번호 bcrypt 해싱
- 로그인 시도 제한
- 보안 확장 규칙: 미적용 (MVP 단계)

### NFR-03: 사용성
- 터치 친화적 UI (최소 44x44px 버튼)
- 카드 형태 메뉴 레이아웃
- 명확한 시각적 계층 구조
- QR 코드 + 태블릿 키오스크 모드 지원

### NFR-04: 테스트
- 단위 테스트 + 통합 테스트
- 속성 기반 테스트(PBT): 미적용

### NFR-05: 배포
- 로컬/온프레미스 서버 배포
- 모노레포 구조 (프론트엔드 + 백엔드 동일 저장소)

## 4. 데이터 모델 (개요)

### 핵심 엔티티
- **Store**: 매장 정보
- **Admin**: 관리자 계정 (역할: 매니저/직원)
- **Table**: 테이블 정보 및 세션 관리
- **TableSession**: 테이블 세션 (첫 주문 ~ 이용 완료)
- **Category**: 메뉴 카테고리
- **MenuItem**: 메뉴 항목
- **Order**: 주문 정보
- **OrderItem**: 주문 상세 항목
- **OrderHistory**: 과거 주문 이력

## 5. 제외 기능 (MVP 범위 외)

- 결제 처리 (카드, 현금, PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 복잡한 인증 (OAuth, SNS 로그인, 2FA)
- 이미지 업로드/리사이징 (MVP에서는 URL만)
- 알림 시스템 (푸시, SMS, 이메일, 소리/진동)
- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (데이터 분석, 매출 리포트, 직원 관리, 예약, 리뷰, 다국어)
- 외부 연동 (배달 플랫폼, POS, 소셜 미디어, 지도, 번역 API)
